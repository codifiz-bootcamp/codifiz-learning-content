Why This Matters

The command line (also called terminal, shell, or console) is how developers interact with their computers at a deeper level. While it looks intimidating at first, it's actually faster and more powerful than clicking through folders. Every professional developer uses the command line daily for tasks like running code, managing files, using Git, and deploying applications.

Fun fact: The command line hasn't changed much in 40+ years because it's so efficient! Once you learn it, these skills transfer everywhere.

1 Opening Your Terminal

The terminal is a text-based interface to your computer. Let's find and open it!

Opening Terminal on Windows

Windows has multiple terminal options. We'll use Command Prompt or PowerShell:

Method 1: Search

  1. Press Windows Key

    Type "cmd" or "powershell"

  2. Click the Result

    Select "Command Prompt" or "Windows PowerShell"

Method 2: Keyboard Shortcut

Press Windows + R, type cmd, and press Enter

Method 3: VS Code Terminal

Open VS Code and press Ctrl + ` (backtick key, usually above Tab)

Recommendation: For this bootcamp, use Git Bash (installed with Git yesterday) or the VS Code integrated terminal. They provide a Unix-like experience that matches what you'll use professionally.

Opening Terminal on Mac

Method 1: Spotlight Search

  1. Press Command + Space

    This opens Spotlight Search

  2. Type "Terminal"

    Press Enter when you see Terminal.app

Method 2: Applications Folder

Go to Applications ‚Üí Utilities ‚Üí Terminal

Method 3: VS Code Terminal

Open VS Code and press Ctrl + ` (backtick key)

Pro Tip: Drag Terminal.app to your Dock for quick access. You'll use it constantly!

Opening Terminal on Linux

Method 1: Keyboard Shortcut

Press Ctrl + Alt + T (works on most Linux distributions)

Method 2: Application Menu

Look for "Terminal" in your applications menu

Method 3: VS Code Terminal

Open VS Code and press Ctrl + ` (backtick key)

Verify Your Terminal is Open

You should see a window with text that looks something like:

C:\Users\YourName>           (Windows)
yourname@computer:~$         (Mac/Linux)

This is called the prompt. It's waiting for you to type a command!

What the Symbols Mean

  • ~ (tilde) - Your home directory (like C:\Users\YourName or /Users/yourname)
  • $ - Regular user prompt (you have normal permissions)
  • # - Admin/root prompt (you have elevated permissions - be careful!)
  • / - Directory separator (on Mac/Linux and Git Bash)
  • \ - Directory separator (on Windows CMD/PowerShell)

Your First Command: Where Am I?

Type this command and press Enter:

pwd

pwd stands for "Print Working Directory" - it shows you exactly where you are in your file system.

You'll see output like:

/Users/yourname              (Mac/Linux)
C:\Users\YourName            (Windows)

You Just Ran Your First Command!

See? Not so scary! You just asked your computer "where am I?" and it told you.

3 Essential Navigation Commands

Let's learn the commands you'll use every single day as a developer.

Command #1: ls (List)

Shows you what files and folders are in your current location.

# Basic list
ls

# List with details (size, date, permissions)
ls -l

# List all files (including hidden ones starting with .)
ls -a

# List all with details (most useful!)
ls -la

# Basic list
dir

# List with details
dir /w

# List all files including hidden
dir /a

Good News: If you're using Git Bash, you can use the Mac/Linux commands (ls) instead of dir!

Try it now: Type ls (or dir) and press Enter. You should see a list of files and folders!

Command #2: cd (Change Directory)

Moves you from one folder to another. Think of it as "clicking into a folder" but with text.

# Go into a folder
cd FolderName

# Go up one level (to parent directory)
cd ..

# Go up two levels
cd ../..

# Go to your home directory
cd ~              # Mac/Linux
cd %USERPROFILE%  # Windows CMD
cd $HOME          # PowerShell

# Go to Desktop
cd Desktop

# Go to a specific path
cd /Users/yourname/Documents        # Mac/Linux
cd C:\Users\YourName\Documents      # Windows

Important Notes

  • Case matters! cd Desktop is different from cd desktop
  • Spaces matter! For folders with spaces, use quotes: cd "My Documents"
  • .. means "parent directory" (one level up)
  • . means "current directory"

Practice Exercise: Navigate Around

4 Creating and Managing Files

Now let's create, move, copy, and delete files - all from the command line!

Command #3: mkdir (Make Directory)

Creates a new folder.

# Create one folder
mkdir bootcamp-practice

# Create multiple folders at once
mkdir folder1 folder2 folder3

# Create nested folders (Mac/Linux/Git Bash)
mkdir -p projects/week1/day1

# Create nested folders (Windows CMD)
mkdir projects\week1\day1

Command #4: touch / new item (Create Files)

# Create a single file
touch index.html

# Create multiple files at once
touch file1.txt file2.txt file3.txt

# Create file in a specific folder
touch projects/readme.md

# Create a single file
type nul > index.html

# Create multiple files (use multiple commands)
type nul > file1.txt
type nul > file2.txt
type nul > file3.txt

# Or use echo
echo. > index.html

Command #5: cp (Copy)

# Copy a file
cp original.txt copy.txt

# Copy to a different folder
cp file.txt ../other-folder/

# Copy a folder and all its contents
cp -r folder-name copy-of-folder

# Copy a file
copy original.txt copy.txt

# Copy to a different folder
copy file.txt ..\other-folder\

# Copy a folder and all contents
xcopy folder-name copy-of-folder /E /I

Command #6: mv (Move/Rename)

# Rename a file
mv oldname.txt newname.txt

# Move a file to another folder
mv file.txt ../documents/

# Move and rename at the same time
mv file.txt ../documents/newname.txt

# Rename a file
ren oldname.txt newname.txt

# Move a file to another folder
move file.txt ..\documents\

# Move and rename
move file.txt ..\documents\newname.txt

Command #7: rm (Remove/Delete)

DANGER ZONE!

Files deleted with rm are GONE FOREVER - they don't go to trash/recycle bin. There's no undo! Always double-check before deleting.

# Delete a single file
rm filename.txt

# Delete multiple files
rm file1.txt file2.txt

# Delete an empty folder
rmdir foldername

# Delete a folder and everything in it (CAREFUL!)
rm -rf foldername

# -r = recursive (includes contents)
# -f = force (no confirmation)

# Delete a single file
del filename.txt

# Delete multiple files
del file1.txt file2.txt

# Delete an empty folder
rmdir foldername

# Delete a folder and everything in it
rmdir /s foldername

# /s = recursive
# /q = quiet (no confirmation)

7 Bonus: View File Contents

Sometimes you need to quickly check what's inside a file without opening an editor.

Command #8: cat / type (Display File Contents)

# Display entire file
cat filename.txt

# Display with line numbers
cat -n filename.txt

# Display multiple files
cat file1.txt file2.txt

# Create a new file with content
cat > newfile.txt
# Type content, then press Ctrl+D to save

# Display entire file
type filename.txt

# Display multiple files
type file1.txt file2.txt

# In PowerShell, you can also use:
Get-Content filename.txt

Command #9: echo (Print Text)

Display text or create simple text files.

# Print text to terminal
echo "Hello, World!"

# Create a file with content (Mac/Linux/Git Bash)
echo "# My Project" > README.md

# Append to a file (Mac/Linux/Git Bash)
echo "More content" >> README.md

# Print a variable (we'll learn this later)
echo $PATH

Quick Reference: > vs >>

  • > - Creates new file or overwrites existing file
  • >> - Appends to existing file (or creates if doesn't exist)
