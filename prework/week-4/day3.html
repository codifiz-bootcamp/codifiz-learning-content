Why This Matters

Git is version control - think of it as "Track Changes" for your code, but infinitely more powerful. It lets you save snapshots of your work, experiment without fear, and collaborate with other developers.

GitHub is where developers live. Over 100 million developers use GitHub to store, share, and collaborate on code. Your GitHub profile is essentially your developer resume. Every job application will ask for your GitHub username!

Together, Git + GitHub = The most important tools in modern software development. üöÄ

2 Configure Git

Tell Git who you are. This information will be attached to every commit you make!

Set Your Name and Email

# Set your name (use your real name)
git config --global user.name "Your Full Name"

# Set your email (use the same email you'll use for GitHub!)
git config --global user.email "youremail@example.com"

Important!

  • Replace "Your Full Name" with your actual name
  • Use the SAME email you'll register with GitHub
  • This email will be visible on GitHub - use a professional one

Verify Your Configuration

# View all Git settings
git config --list

# Or check specific values
git config user.name
git config user.email

Optional: Set Default Branch Name

GitHub now uses "main" as the default branch name (instead of "master"):

git config --global init.defaultBranch main

Step 5.4: Initialize Git

Back in your terminal (inside the my-first-repo folder):

# Initialize Git in this folder

This means Git sees README.md but isn't tracking it yet (shown in red).

6 Understanding the Git Workflow

Git has a three-step workflow. Understanding this is CRUCIAL!

The Git Workflow

Working Directory ‚Üí Staging Area ‚Üí Repository
(Your Files) ‚Üí (git add) ‚Üí (git commit) ‚Üí (git push) ‚Üí GitHub
  1. Working Directory: Your actual files that you edit
  2. Staging Area: Files you've marked to be saved (with git add)
  3. Repository: Saved snapshots of your code (with git commit)
  4. GitHub: Your code backed up online (with git push)

Step 6.1: Stage Your File

"Staging" means telling Git "I want to save this file."

# Stage a single file
git add README.md

# Or stage all changed files
git add .

# Check status again
git status

Now README.md should be green! It's staged and ready to commit.

Step 6.2: Commit Your Changes

A "commit" is like taking a snapshot. You can always come back to this exact version later!

git commit -m "Initial commit: Add README with project description"

Commit Message Best Practices

Good commit messages:

  • ‚úÖ "Add README with project description"
  • ‚úÖ "Fix login button alignment"
  • ‚úÖ "Update homepage hero section"

Bad commit messages:

  • ‚ùå "stuff"
  • ‚ùå "changes"
  • ‚ùå "asdfgh"
  • ‚ùå "idk what I did"

Format: Start with a verb, be specific, use present tense

Step 6.3: View Your Commit History

# See all commits
git log

# See commits in one line each (cleaner)
git log --oneline

7 Push to GitHub

Now let's put your code on GitHub so the world can see it!

Step 7.1: Create a Repository on GitHub

  1. Go to GitHub.com

    Make sure you're logged in

  2. Click the "+" Icon

    Top-right corner ‚Üí "New repository"

  3. Fill Out Repository Details
    • Repository name: my-first-repo (match your local folder name)
    • Description: "My first GitHub repository for bootcamp pre-work"
    • Public or Private: Choose Public (so employers can see it)
    • Initialize repository: DO NOT check any boxes (we already have files)
  4. Click "Create repository"

    You'll see instructions on the next page

Step 7.2: Connect Local to GitHub

GitHub will show you commands. You need the section: "‚Ä¶or push an existing repository from the command line"

Back in your terminal:

# Add GitHub as the remote
git remote add origin https://github.com/YOUR-USERNAME/my-first-repo.git

# Verify the remote was added
git remote -v

Replace YOUR-USERNAME

Make sure to replace YOUR-USERNAME with your actual GitHub username!

Step 7.3: Rename Branch to Main

git branch -M main

Step 7.4: Push Your Code

git push -u origin main

You'll be prompted for credentials. See next section!

Step 7.5: GitHub Authentication

GitHub no longer accepts passwords for Git operations. You need a Personal Access Token.

Create a Personal Access Token

  1. On GitHub, click your profile icon ‚Üí Settings
  2. Scroll to bottom of left sidebar ‚Üí Developer settings
  3. Click Personal access tokens ‚Üí Tokens (classic)
  4. Click Generate new token ‚Üí Generate new token (classic)
  5. Note: "Bootcamp access token"
  6. Expiration: 90 days (or custom)
  7. Select scopes: Check repo (this checks all sub-items automatically)
  8. Click Generate token
  9. COPY THE TOKEN NOW! (You won't see it again)
  10. Save it in a password manager

When prompted for password, paste your token (not your GitHub password!)

Step 7.6: Verify on GitHub

Go to https://github.com/YOUR-USERNAME/my-first-repo

8 The Complete Git Workflow

Now that you've done it once, here's the workflow you'll use every day:

# 1. Check what changed
git status

# 2. Stage your changes
git add .                    # Stage all files
git add filename.txt         # Stage specific file

# 3. Commit with a message
git commit -m "Describe what you changed"

# 4. Push to GitHub
git push

# That's it! Repeat as needed.

Essential Git Commands Reference

Command What It Does
git init Create a new Git repository
git status Check what files have changed
git add . Stage all changed files
git add filename Stage a specific file
git commit -m "message" Save a snapshot with a message
git push Upload commits to GitHub
git pull Download changes from GitHub
git log View commit history
git log --oneline View compact commit history
git remote -v View connected GitHub repositories
