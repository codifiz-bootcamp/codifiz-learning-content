Why This Matters

Node.js allows JavaScript to run outside the browser‚Äîon servers, in build tools, and more. It's essential for modern web development because it lets you use JavaScript for EVERYTHING (frontend and backend).

npm (Node Package Manager) gives you access to over 2 million code libraries. Instead of building everything from scratch, you can use packages created by millions of developers worldwide. It's like having an app store for code!

Fun fact: npm is the largest software registry in the world. Companies like Netflix, LinkedIn, and NASA use npm packages in production! üöÄ

1 Install Node.js

Node.js includes npm automatically, so installing Node.js gives you both!

Install Node.js on Windows

  1. Download Node.js

    Go to nodejs.org

    Download the LTS version (Long Term Support - left button)

    LTS vs Current:

    • ‚úÖ LTS (Recommended): Stable, reliable, used in production
    • ‚ùå Current: Latest features but may be unstable
  2. Run the Installer

    Open the downloaded .msi file

    Click "Next" ‚Üí Accept license ‚Üí "Next" ‚Üí "Next" ‚Üí "Install"

    Accept all default settings

  3. Complete Installation

    Click "Finish"

    Restart your terminal if it was open

Install Node.js on Mac

  1. Download Node.js

    Go to nodejs.org

    Download the LTS version (left button)

  2. Run the Installer

    Open the downloaded .pkg file

    Follow the installation wizard

    You may need to enter your password

  3. Complete Installation

    Close and reopen Terminal

Install Node.js on Linux

Ubuntu/Debian (Recommended Method):

# Install Node.js LTS via NodeSource
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt-get install -y nodejs

Fedora/RHEL:

sudo dnf install nodejs

Arch Linux:

sudo pacman -S nodejs npm

Open your terminal and run these commands:

# Check npm version

You should see something like:

2 Your First Node.js Program

Let's run JavaScript outside the browser for the first time!

Method 1: Node REPL (Read-Eval-Print Loop)

The REPL lets you run JavaScript commands interactively.

# Start Node REPL
node

You'll see a > prompt. Try these commands:

> console.log("Hello from Node.js!")
Hello from Node.js!

> 2 + 2
4

> const name = "Developer"
undefined

> name.toUpperCase()
'DEVELOPER'

> const sum = (a, b) => a + b
undefined

> sum(5, 3)
8

> .exit  // Exit REPL (or press Ctrl+C twice)

Method 2: Run a JavaScript File

Let's create a JavaScript file and run it with Node!

# Navigate to Desktop
cd Desktop

# Create a new file
touch hello.js        # Mac/Linux/Git Bash
type nul > hello.js   # Windows CMD

# Open in VS Code
code hello.js

Add this code to hello.js:

// My first Node.js program!
console.log("Hello from Node.js!");

const greet = (name) => {
  return `Hello, ${name}! Welcome to Node.js!`;
};

console.log(greet("Developer"));

// Do some math
const add = (a, b) => a + b;
const result = add(10, 5);
console.log(`10 + 5 = ${result}`);

Save the file and run it:

node hello.js

You should see:

Hello from Node.js!
Hello, Developer! Welcome to Node.js!
10 + 5 = 15

Congratulations!

You just ran JavaScript on your computer (not in a browser)! This is HUGE - you can now build servers, CLI tools, automation scripts, and more!

3 Initialize Your First npm Project

npm projects use a package.json file to track information about your project and its dependencies.

Create a Project Folder

# Create project folder
mkdir npm-practice
cd npm-practice

# Verify you're in the right place
pwd

Initialize npm

# Initialize with defaults (fastest)
npm init -y

# OR initialize with prompts (interactive)
npm init

The -y flag says "yes" to all defaults. This creates a package.json file!

Understanding package.json

Open the newly created package.json:

{
  "name": "npm-practice",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],
  "author": "",
  "license": "ISC"
}

What Each Field Means:

  • name: Your project name (must be lowercase, no spaces)
  • version: Project version (follows semantic versioning)
  • description: What your project does
  • main: Entry point file
  • scripts: Commands you can run with npm run
  • dependencies: Packages your project needs (added when you install packages)
  • author: Your name
  • license: How others can use your code

4 Installing npm Packages

Let's install a popular package called chalk that adds colors to terminal output!

Install Your First Package

# Install chalk (version 4.x for CommonJS compatibility)
npm install chalk@4

This command does several things:

  1. Downloads chalk and its dependencies
  2. Creates a node_modules folder with the package code
  3. Updates package.json with the dependency
  4. Creates package-lock.json (locks exact versions)

Important: node_modules

The node_modules folder can be HUGE (thousands of files!).

  • ‚ùå Never edit files in node_modules directly
  • ‚ùå Never commit node_modules to Git
  • ‚úÖ Always commit package.json and package-lock.json
  • ‚úÖ Use .gitignore to exclude node_modules

Using an Installed Package

Create index.js:

touch index.js        # Mac/Linux/Git Bash
type nul > index.js   # Windows CMD

code index.js

Add this code:

// Import the chalk package
const chalk = require('chalk');

// Use it to add colors!
console.log(chalk.blue('Hello in blue!'));
console.log(chalk.red.bold('Error message in red and bold!'));
console.log(chalk.green.underline('Success!'));
console.log(chalk.yellow('Warning: This is amazing!'));
console.log(chalk.cyan.bgBlack(' Black background with cyan text '));

// You can also use hex colors
console.log(chalk.hex('#FF69B4')('Hot pink text!'));

Run it:

node index.js

You should see beautiful colored output in your terminal! üé®

You Just Used Your First npm Package!

This is exactly how professional developers work - using packages to avoid reinventing the wheel!

5 Create .gitignore

Before pushing to GitHub, we need to tell Git to ignore node_modules.

# Create .gitignore file
touch .gitignore        # Mac/Linux/Git Bash
type nul > .gitignore   # Windows CMD

# Open it
code .gitignore

Add this content:

# Dependencies
node_modules/

# Environment variables
.env
.env.local

# Operating System files
.DS_Store
Thumbs.db

# Editor directories
.vscode/
.idea/

# Logs
*.log
npm-debug.log*

# Build output
dist/
build/

Save the file. Now Git will ignore all these files and folders!

Why This Matters:

If you git add . without .gitignore, you'd commit thousands of node_modules files to GitHub. That would:

  • Make your repository HUGE (hundreds of MB)
  • Slow down cloning and pushing
  • Look unprofessional

Instead, others can just run npm install to get all dependencies!

6 npm Scripts

npm scripts let you create shortcuts for commands you run often.

Edit package.json

Open package.json and update the scripts section:

{
  "name": "npm-practice",
  "version": "1.0.0",
  "description": "Learning npm package management",
  "main": "index.js",
  "scripts": {
    "start": "node index.js",
    "test": "echo \"Running tests...\"",
    "dev": "node index.js",
    "hello": "echo \"Hello from npm scripts!\""
  },
  "author": "Your Name",
  "license": "ISC",
  "dependencies": {
    "chalk": "^4.1.2"
  }
}

Run npm Scripts

# Run the start script
npm start

# Run the test script
npm test

# Run custom scripts (need "run")
npm run dev
npm run hello

npm Script Shortcuts:

  • npm start - Doesn't need "run" (special script)
  • npm test - Doesn't need "run" (special script)
  • npm run SCRIPT_NAME - All custom scripts need "run"

Essential npm Commands Reference

Command What It Does
