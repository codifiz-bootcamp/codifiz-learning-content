1 What Are AI Coding Assistants?

AI coding assistants are tools powered by large language models (LLMs) that help you write code.

How They Work

  1. Context Awareness: They read your code, comments, and file names
  2. Pattern Recognition: They've learned from billions of lines of public code
  3. Smart Suggestions: They predict what you're trying to write next
  4. On-Demand Help: You can ask questions and get explanations

2 Setting Up GitHub Copilot

GitHub Copilot integrates directly into VS Code and suggests code as you type.

Step 1: Get GitHub Copilot Access

Students Get Copilot FREE!

  1. Apply for GitHub Student Developer Pack

    Go to education.github.com/pack

  2. Verify Your Student Status

    Upload proof of enrollment (student ID, transcript, or school email)

  3. Get Approved

    Usually takes 1-2 days. You'll get an email when approved.

  4. Benefits Include:
    • GitHub Copilot (FREE - normally $10/month)
    • GitHub Pro (unlimited private repos)
    • Free domain name for 1 year
    • $100+ in cloud credits
    • Many more developer tools!

Paid Copilot Plan

  1. Go to github.com/features/copilot
  2. Click "Start my free trial" (30 days free)
  3. After trial: $10/month for individuals

Step 2: Install Copilot Extension in VS Code

  1. Open VS Code Extensions

    Click the Extensions icon (or press Ctrl+Shift+X / Cmd+Shift+X)

  2. Search for "GitHub Copilot"

    Type "GitHub Copilot" in the search box

  3. Install Both Extensions:
    • GitHub Copilot - The main extension
    • GitHub Copilot Chat - For conversational coding help
  4. Sign In

    Click "Sign in to GitHub" in the popup and authorize the extension

Step 3: Verify Copilot is Working

Quick Test

  1. Create a new file called test.js
  2. Type this comment:
// function to calculate the area of a circle
  1. Press Enter and wait 1-2 seconds
  2. Copilot should suggest code in gray text!
  3. Press Tab to accept the suggestion

üéâ It's Working!

If you see gray text suggestions, Copilot is active! If not, check that you're signed in and have Copilot access.

3 Installing Cursor AI

Cursor is an AI-first code editor built on VS Code. It's designed for AI-assisted development from the ground up.

Why Use Cursor?

  • Chat with your codebase: Ask questions about how your code works
  • AI-powered editing: Select code and ask AI to modify it
  • Composer mode: AI can edit multiple files at once
  • Built-in terminal AI: Get command suggestions
  • Free tier available: 50 AI requests per day on free plan

Install Cursor

  1. Go to cursor.com
  2. Click "Download for Windows"
  3. Run the installer (Cursor Setup.exe)
  4. Follow installation wizard
  5. Launch Cursor
  1. Go to cursor.com
  2. Click "Download for Mac"
  3. Open the .dmg file
  4. Drag Cursor to Applications folder
  5. Launch Cursor from Applications
  1. Go to cursor.com
  2. Download the AppImage or .deb file
  3. Make it executable:
chmod +x Cursor-*.AppImage
./Cursor-*.AppImage

First-Time Setup

  1. Sign In or Skip

    You can sign in with GitHub/Google or skip for now

  2. Import VS Code Settings (Optional)

    Cursor can import your VS Code extensions and settings

  3. Choose AI Model

    Go to Settings ‚Üí Models ‚Üí Choose Claude, GPT-4, or others

  4. Try the AI Chat

    Press Cmd+L (Mac) or Ctrl+L (Windows/Linux) to open AI chat

4 Writing Effective AI Prompts

AI assistants are powerful, but the quality of their output depends on the quality of your input. Here's how to write great prompts:

The CLEAR Prompting Framework

  • Context: What are you building? What's the purpose?
  • Language/Stack: What technologies should be used?
  • Expectations: What should the code do?
  • Actions: What specific task needs to be done?
  • Requirements: Any constraints or specific requirements?

Examples: Bad vs Good Prompts

‚ùå Bad Prompt ‚úÖ Good Prompt 
// make a function
 
// Create a JavaScript function that validates email addresses
// Should return true for valid emails, false otherwise
// Must handle edge cases like multiple @ signs
 
// login page
 
// Create a React login page component with:
// - Email and password input fields
// - Form validation
// - Submit button
// - Error message display
// - Responsive design using Tailwind CSS
 
// fix this
 
// This function should sort an array of numbers in ascending order
// but it's not working correctly. Please fix the logic error.
// Expected: [1, 2, 3, 4, 5]
// Getting: [1, 3, 2, 5, 4]

Comment-Driven Development

Write comments describing what you want, then let AI fill in the code!

// TODO: Create a function called fetchUserData
// - Takes a userId parameter
// - Fetches data from the API at https://api.example.com/users/{userId}
// - Returns the user object
// - Handles errors gracefully with try/catch
// - Shows a loading state while fetching

Now press Enter and let Copilot suggest the implementation!

Cursor Chat Best Practices

How to Ask Questions in Cursor:

  • "Explain this code" - Select code and ask for explanation
  • "Add error handling to this function" - Improves existing code
  • "Write tests for this component" - Generates test cases
  • "Refactor this to use async/await" - Modernizes code
  • "What's wrong with this code?" - Debugging help
  • "How do I deploy this to Vercel?" - Deployment guidance

Best Practices for AI-Assisted Coding

‚úÖ DO:

  • Write clear, detailed comments describing what you need
  • Review and understand all AI-generated code
  • Test thoroughly before deploying
  • Use AI to learn new concepts and patterns
  • Ask AI to explain code you don't understand
  • Iterate on suggestions - ask for improvements
  • Use AI for boilerplate and repetitive code

‚ùå DON'T:

  • Blindly copy-paste without understanding
  • Trust AI 100% - it makes mistakes
  • Skip learning fundamentals
  • Use AI-generated code for security-critical functions without review
  • Assume AI suggestions are the best solution
  • Forget to test edge cases
  • Rely on AI for production code without human oversight
