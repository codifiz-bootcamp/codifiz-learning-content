Day 1: HTML Basics & Document Structure

3 hours 6 objectives Beginner

Why This Matters

HTML (HyperText Markup Language) is the foundation of every website on the internet. It's the skeleton that gives structure to web pages. Before you can build beautiful, interactive web applications, you need to understand how to structure content with HTML. Think of HTML as the blueprint of a house‚Äîwithout it, you can't add the paint (CSS) or the electrical systems (JavaScript).

Real-world impact: Every web developer, regardless of specialization, uses HTML daily. Even modern frameworks like React ultimately generate HTML. Mastering HTML fundamentals is essential for your journey as a full-stack developer.

Learning Objectives

What is HTML?

HTML stands for HyperText Markup Language. Let's break that down:

  • HyperText: Text that contains links to other texts (the "hyper" means "beyond" or "above")
  • Markup: A system of annotating text to indicate how it should be displayed
  • Language: A set of rules and syntax that browsers understand

Simple Analogy

Think of HTML like the structure of a newspaper article:

  • The headline is like an <h1> tag
  • Paragraphs are like <p> tags
  • Lists of bullet points are like <ul> and <li> tags
  • Images are like <img> tags

The Basic HTML Document Structure

Every HTML document follows a standard structure. Here's the fundamental skeleton that every webpage needs:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is my first HTML page.</p>
</body>
</html>

Understanding Each Part

1. <!DOCTYPE html>

This declaration tells the browser that this document is an HTML5 document. It must be the very first line of your HTML file. Think of it as saying "Hey browser, I'm speaking HTML5!"

2. <html lang="en">

The root element that wraps all content on the page. The lang="en" attribute specifies the language (English) for accessibility tools and search engines.

3. <head>

Contains metadata about the document‚Äîinformation that's not displayed on the page but is important for browsers and search engines:

  • <meta charset="UTF-8"> - Character encoding for the document
  • <meta name="viewport"...> - Makes your page responsive on mobile devices
  • <title> - The text that appears in the browser tab

4. <body>

Contains all the visible content of your webpage‚Äîeverything users actually see and interact with.

Essential HTML Elements

Let's explore the most commonly used HTML elements you'll need to know:

Headings

HTML has six levels of headings, from <h1> (most important) to <h6> (least important):

<h1>Main Heading - Use Only Once Per Page</h1>
<h2>Section Heading</h2>
<h3>Subsection Heading</h3>
<h4>Smaller Heading</h4>
<h5>Even Smaller</h5>
<h6>Smallest Heading</h6>

Best Practice

Use only one <h1> per page (typically your page title). Don't skip heading levels (don't go from h1 to h3). Think of headings like an outline or table of contents.

Paragraphs

The <p> element represents a paragraph of text:

<p>This is a paragraph. It can contain as much text as you want.</p>

<p>This is another paragraph. Browsers automatically add space 
   between paragraphs.</p>

Lists

There are two main types of lists in HTML:

<!-- Unordered List (Bullet Points) -->
<ul>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ul>

<!-- Ordered List (Numbered) -->
<ol>
    <li>First step</li>
    <li>Second step</li>
    <li>Third step</li>
</ol>

Links

The <a> (anchor) element creates hyperlinks:

<!-- External Link -->
<a href="https://www.google.com">Visit Google</a>

<!-- Link that opens in new tab -->
<a href="https://www.github.com" target="_blank">Visit GitHub</a>

<!-- Link to another page in your site -->
<a href="about.html">About Page</a>

<!-- Email link -->
<a href="mailto:hello@example.com">Email Me</a>

Images

The <img> element displays images:

<!-- Image with alt text for accessibility -->
<img src="photo.jpg" alt="A beautiful sunset">

<!-- Image with width and height -->
<img src="logo.png" alt="Company Logo" width="200" height="100">

<!-- Image from URL -->
<img src="https://via.placeholder.com/300" alt="Placeholder image">

Accessibility Matters

Always include the alt attribute on images. This text describes the image for screen readers (used by visually impaired users) and displays if the image fails to load.

Text Formatting

Common elements for formatting text:

<p>This is <strong>bold text</strong> (important).</p>
<p>This is <em>italic text</em> (emphasized).</p>
<p>This is <mark>highlighted text</mark>.</p>
<p>This is <del>deleted text</del>.</p>
<p>This is <ins>inserted text</ins>.</p>
<p>This is <small>smaller text</small>.</p>

Semantic HTML

Semantic HTML uses elements that clearly describe their meaning to both the browser and the developer. Instead of using generic <div> elements for everything, we use semantic elements that describe the content's purpose.

Why Semantic HTML Matters:

  • SEO: Search engines better understand your content
  • Accessibility: Screen readers can navigate your page more effectively
  • Maintainability: Code is easier to read and understand
  • Consistency: Standard elements everyone recognizes

Common Semantic Elements

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Semantic HTML Example</title>
</head>
<body>
    <header>
        <h1>My Website</h1>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#about">About</a></li>
                <li><a href="#contact">Contact</a></li>
            </ul>
        </nav>
    </header>

    <main>
        <article>
            <h2>Article Title</h2>
            <p>Article content goes here...</p>
        </article>

        <section>
            <h2>Section Title</h2>
            <p>Section content...</p>
        </section>

        <aside>
            <h3>Related Links</h3>
            <ul>
                <li><a href="#">Link 1</a></li>
                <li><a href="#">Link 2</a></li>
            </ul>
        </aside>
    </main>

    <footer>
        <p>© 2025 My Website. All rights reserved.</p>
    </footer>
</body>
</html>

Element Definitions:

  • <header> - Top section with site branding and navigation
  • <nav> - Navigation links
  • <main> - Main content area (use only once per page)
  • <article> - Self-contained content (blog post, news article)
  • <section> - Thematic grouping of content
  • <aside> - Content tangentially related to main content (sidebars)
  • <footer> - Bottom section with copyright, links, contact info

Common HTML Errors and How to Fix Them

Error 1: Forgetting to Close Tags

‚ùå Wrong:

<p>This is a paragraph
<p>This is another paragraph

‚úÖ Correct:

<p>This is a paragraph</p>
<p>This is another paragraph</p>

How to prevent: VS Code automatically closes tags for you. Pay attention to the color coding‚Äîunclosed tags often appear in a different color.

Error 2: Nesting Tags Incorrectly

‚ùå Wrong:

<p>This is <strong>bold text</p></strong>

‚úÖ Correct:

<p>This is <strong>bold text</strong></p>

Rule: The last tag opened must be the first tag closed. Think of it like parentheses in math.

Error 3: Missing Quotes on Attributes

‚ùå Wrong:

<img src=photo.jpg alt=My Photo>

‚úÖ Correct:

<img src="photo.jpg" alt="My Photo">

Best practice: Always use double quotes around attribute values.

Error 4: Multiple <h1> Tags

‚ö†Ô∏è Not Recommended:

<h1>Welcome</h1>
<h1>About Us</h1>
<h1>Contact</h1>

‚úÖ Better:

<h1>Welcome to Our Site</h1>
<h2>About Us</h2>
<h2>Contact</h2>

Why: Search engines and screen readers expect one main heading per page.

Self-Check Quiz

Test your understanding of HTML basics. Try to answer without looking back!

1. What does HTML stand for?

Answer: b) HyperText Markup Language

HTML is the standard markup language for creating web pages. "HyperText" refers to links that connect web pages, and "Markup" describes the way tags are used to define elements.

2. Which of the following is the correct HTML document structure?

Answer: b) <html><head></head><body></body></html>

The correct structure has the <html> element wrapping everything, with <head> (metadata) coming before <body> (visible content).

3. What is the purpose of the <head> element?

Answer: b) To contain metadata and links to stylesheets

The <head> section contains information about the document that isn't displayed on the page, like the title, character encoding, and links to CSS files.

4. Why is semantic HTML important?

Answer: b) It improves accessibility, SEO, and code readability

Semantic HTML uses elements that clearly describe their meaning. This helps search engines understand your content, makes your site more accessible to users with disabilities, and makes your code easier for developers to understand.

5. What's the difference between <strong> and <b> tags?

Answer: b) <strong> indicates importance (semantic), <b> is just visual

While both make text bold, <strong> is semantic and indicates that the text is important, which screen readers will emphasize. The <b> tag is purely presentational and doesn't convey any special meaning.
